Introduction

HTML is the standard markup language for webpages used to structure a web page and its contents.

HTML is made up of a collection of elements or tags, that wrap around different parts of content on a page which allow content to appear or behave a certain way.

<html>...</html>

Tags can make text appear as a large heading or as a link to another page. They can create bulleted lists or even display an image.

This technical document will go over the basics of HTML.

Basic Structure

The basic structure of a HTML file is as follows,

<!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta http-equiv="X-UA-Compatible" content="IE=edge">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Document</title>
    </head>
    <body>
    </body>
</html>

This is how all HTML documents start. A <head> and <body> tag pair within a <html> tag pair

The head tag is used to contain metadata about the page such as what character set the page is usingf, device information and compatibility information. Metadata is not displayed.

The head also contains link tags that allow us to link stylesheets (CSS), for the visual styling of the page and javascript file (JS), for interaction on the page.

Head

The <head> element goes before the body element within the <html> element. The head of the document contains all of the metadata, links to external css and JS files and the all important page <title> element.

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="sylesheet" href="styles.css">
    <script src="script.js">
    <title>Document</title>
</head>
  • meta - The meta tag allows us to provide document metadata that cannot be represented by other elements. Such as charset, the documents character encoding. Or the device width details.
  • link - The link element specifies relationships between the documents it is added to and an external recourse. THe main example is an external CSS file. THe link element allows us to link the style sheet to the document so they can be physically separated. We can also link external icon sets or even fonts such as Google fonts.
  • script - The script element allows us to embed execuatable code such as JavaScript. The code can either be entered between the tags in the HTML doc, or the src attribute can be used to link an external JS document, similar to how the link element works for a CSS file.
  • title - The title element allows us to provide a title for the page. This served a few purposes. Firstly it will display this title in the browsers title bar and browser tab. Secondly, it plays a role in Search Engine Optimisation (SEO) a good, descriptive title helps with SEO.

Body

All the content that you see in the browser tab viewport is stored between the tags of the <body> element, there can only be one of these elements in a document. The use of general divider/sections/container based elements is used to create the overall layout of the page as shown below.

Tags

Tags are the building blocks of web pages, there are no web pages without them. Below shows the structrure of a tag.

There are numerous tags that can be used in HTML, all serving a specific purpose a few of which are detailed below.

  • <p> - This is the paragraph element, one of the basic and most used elements for structring text.
  • <div> - This the <div> element. A <div> defines a division or a section of in a HTML document. It is best to think of a <div> as a container for other elements to be stored which can be easily accessed and styled with CSS or manipulated with JS.
  • <h1> - This is the <h1> element. This represents the main heading of a page. Only one of these should be used per page as search engines and screen readers are specifically looking for this elements to determine what the page is about. There is a heirarchy of heading elements (<h2>, <h3>; and so on). These can have multiple of the same tag however, as they will represent sub headings and indicate different sections.
  • <ul> - This the unordered list element. In combination with <li> elements which are nested inside of it, you can create lists of text for a recipe or links and turn them into a navigation bar. In fact, this section of text you are reading is in a <li> element within a <ul> element.
  • <code> - This element is wrapped around a section of text and formats it using your systems defualt monospace font to make it look like terminal text.

Attributes

Attributes are features of elements in HTML. Attributes are values that can configure elements or modify their behaviour in some way. Attributes are available for tags in the head or body of a document.

Some attribute examples are as follows.

  • class - This is the class attribute. All tags with the same class attribute can have their styles modified with a single class selector in CSS.
  • id - The id attribute allows us to single out an element from the rest of the document. An id attribute value must be unique accross the entire document, it cannot be used by more than one element. This allows us to style an individual element amoung a group of the same elemet types, or tie an element to another funcitonaly, such as an input lable to the specific input.
  • maxlength - The maxlength attribute is used for form validation. We can control the maximum number of characters in a textbox with this attribute. If the number is exceeded, the browser will produce an alert, warning the user of the issue.
  • for - The for attribute ties a <label> to an <input> element. The for attribute on the <label> element relates to the id attribute on the <input> element.
  • required - This attribute is another form validation control. It forces the user to input a value and will show an alert message if an attempt is made to submit an incomplete form.
